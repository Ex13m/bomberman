## О проекте

<!-- BORN:PROJECT -->
**Bomberman-клон** — браузерная аркада на сетке. Hot-seat-мультиплеер: два игрока на
одной клавиатуре и одном экране ставят бомбы, рушат стены и стараются взорвать соперника.
Цель — простой, играбельный клон без сборки, открывается двойным кликом по `index.html`.

## Структура папок

| Папка     | Что лежит                                                              |
|-----------|-----------------------------------------------------------------------|
| `kb/`     | База знаний: Markdown-заметки, «один смысловой объект = одна заметка». |
| `data/`   | Внутренние данные проекта в рабочих форматах (`.md` / `.csv` / `.json`). |
| `output/` | Результаты и артефакты на выход (отчёты, выгрузки, готовые файлы).     |
| `tpl/`    | Шаблоны: заготовки заметок, отчётов, писем, конфигов.                  |
| `src/`    | Исходники игры (Vanilla JS, ES-модули): логика, рендер, ввод.          |
| `assets/` | Игровые ассеты: спрайты, звуки, изображения.                           |
| `index.html` | Точка входа — открывается в браузере, подключает `src/main.js`.      |

## Правила именования файлов

- Только `kebab-case`: строчные буквы, слова через `-`, без пробелов и кириллицы в имени.
- Префикс по типу: `kb-` (заметка), `tpl-` (шаблон), `data-` (данные), `out-`
  (результат), `log-` (журнал/лог).
- Дата при версионировании — в формате `ГГГГ-ММ-ДД` внутри имени.

| Тип       | Good ✅                          | Bad ❌                          |
|-----------|----------------------------------|---------------------------------|
| Заметка   | `kb-postgres-indexes.md`         | `Заметка про индексы.md`        |
| Шаблон    | `tpl-weekly-report.md`           | `шаблон отчёта (final).docx`    |
| Данные    | `data-clients-2026-06-20.csv`    | `clients NEW.xlsx`              |
| Результат | `out-summary-2026-06-20.md`      | `итог222.md`                    |

## Форматы данных

| Этап                    | Формат                                                |
|-------------------------|-------------------------------------------------------|
| Хранение внутри проекта | **Markdown / CSV / JSON** — текстовые, диффятся в git.  |
| Вход (импорт)           | Excel / Word / PDF — допустимы только как источник.    |
| Выход (экспорт)         | Excel / Word — только финальная выгрузка в `output/`.  |

Принцип: внутри проекта всё живёт в текстовых форматах; Office-файлы — лишь на границах
(вход и выход). Не хранить рабочие данные в `.xlsx` / `.docx`.

## База знаний (`kb/`)

- Принцип **«один смысловой объект = одна заметка»**: один файл — одна тема/сущность.
- Имя файла = суть заметки в `kebab-case` с префиксом `kb-`.
- Связи между заметками — ссылками `[[kb-имя-заметки]]`, а не дублированием текста.
- Заметка слишком разрослась и охватывает две темы — раздели её на две.

## Как работать со мной

- **Язык ответов — русский.** Технические термины (`commit`, `CLAUDE.md`, `kebab-case`,
  `endpoint` и т.п.) оставляй по-английски, не переводи.
- **Сначала план, потом действие.** Перед массовым переименованием, перемещением или
  удалением файлов — сначала покажи план (что и куда), дождись «ок», затем выполняй.
- Результаты и сгенерированные артефакты складывай **строго в `output/`**.
- Не плодить «вода»-файлы: реши задачу минимумом файлов, переиспользуй существующие.

## Стек и архитектура

- **Vanilla JS + HTML5 Canvas**, ES-модули (`<script type="module">`), без сборки и зависимостей.
- Запуск: открыть `index.html` в браузере (или `run`/локальный статик-сервер для модулей).
- Игра на **сетке (grid)**: мир — двумерный массив клеток (стена / разрушаемый блок / пол).
- Цикл: `requestAnimationFrame` → `update(dt)` → `render(ctx)`. Состояние отделено от рендера.
- Два игрока на одной клавиатуре: P1 — `WASD` + `Space` (бомба), P2 — стрелки + `Enter`.
- Модули в `src/`: `config.js` (константы), `game.js` (состояние/цикл), `player.js`,
  `bomb.js` (бомбы/взрывы), `input.js` (клавиши), `render.js` (отрисовка), `main.js` (вход).

## Definition of Done (играбельный MVP)

- [ ] Рендерится арена-сетка со стенами и разрушаемыми блоками.
- [ ] Оба игрока двигаются своими клавишами, не проходят сквозь стены.
- [ ] Игрок ставит бомбу; через таймер она взрывается крестом по сетке.
- [ ] Взрыв рушит разрушаемые блоки и «убивает» игрока в зоне поражения.
- [ ] Объявляется победитель / ничья, есть рестарт раунда.
